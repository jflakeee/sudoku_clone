# Sudoku Clone - 구현 설계 문서

> 벤치마크: Sudoku.com (스도쿠 클래식 - 로직 퍼즐 게임)
> 플랫폼: 웹 전용 (모바일 기기 대응)
> 작성일: 2026-02-26

---

## 1. 프로젝트 개요

### 1.1 목표
- Sudoku.com 앱의 UI/UX를 웹으로 완전히 재현
- 광고 강제재생 기능만 제외
- 클릭/탭 시 실행되는 모든 기능 구현
- 화면 배치, 애니메이션, 사운드 효과 동일하게 구현

### 1.2 기술 스택
- **프론트엔드**: HTML5 + CSS3 + Vanilla JavaScript (또는 경량 프레임워크)
- **렌더링**: DOM 기반 (Canvas 불필요 - UI 중심 앱)
- **스토리지**: LocalStorage (게임 상태 저장, 통계, 설정)
- **사운드**: Web Audio API
- **PWA**: Service Worker + manifest.json (오프라인 캐시)
- **테스트**: Playwright
- **빌드**: 없음 (정적 파일 배포) 또는 Vite (번들링 필요시)

### 1.3 제외 기능
- 광고 표시/강제 재생
- 인앱 결제
- 서버 사이드 기능 (로그인, 클라우드 동기화)
- iMessage 스티커
- 킬러 스도쿠 프로모션 팝업

## 3. 핵심 게임 로직

### 3.1 스도쿠 퍼즐 생성 알고리즘

```
1. 완성된 유효한 9x9 보드 생성 (Backtracking)
2. 난이도에 따른 셀 제거:
   - 쉬움:   36~40개 제거 (빈 셀)
   - 보통:   41~46개 제거
   - 어려움: 47~51개 제거
   - 전문가: 52~55개 제거
   - 마스터: 56~60개 제거
3. 유일해 검증 (제거 후 해가 하나인지 확인)
```

### 3.2 점수 시스템

영상 분석 기반 점수 규칙:
- 기본 셀 입력: 약 **+250점** (vid3_003에서 확인)
- 게임 완료 시 총 점수 계산
- 최고 점수 기록 관리 (오늘, 이번 주, 이번 달, 통산)

### 3.3 셀 선택 및 하이라이트 규칙

셀 선택 시 하이라이트 적용:
1. **선택된 셀**: 진한 파란색 배경
2. **같은 행**: 연한 파란색 배경
3. **같은 열**: 연한 파란색 배경
4. **같은 3x3 블록**: 연한 파란색 배경
5. **같은 숫자**: 약간 더 진한 파란색 배경 (선택된 셀과 같은 숫자를 가진 모든 셀)

### 3.4 실수 처리

- 실수 한도: 기본 3회 (설정에서 토글 가능)
- 잘못된 숫자 입력 시 실수 카운트 증가
- 실수 표시: `0/3`, `1/3`, `2/3`
- 3회 실수 시 게임 오버

### 3.5 메모 (연필) 모드

- 메모 ON/OFF 토글
- ON 상태에서 숫자 입력 시 작은 숫자로 셀에 표시
- 한 셀에 최대 9개 메모 가능 (3x3 배치)
- 같은 숫자 재탭 시 메모에서 제거
- 확정 숫자 입력 시 관련 행/열/블록의 메모에서 해당 숫자 자동 제거

### 3.6 힌트 시스템

- 스마트 힌트: 스도쿠 풀이 기법을 가르치는 교육적 힌트
  - "마지막 빈 셀": 행/열/블록에 하나만 남았을 때
  - "유일 후보": 특정 셀에 하나의 가능한 숫자만 있을 때
- 힌트 사용 시 셀 자동 채움 또는 하이라이트 안내
- 힌트 횟수 제한 (뱃지로 표시)

### 3.7 실행 취소 (Undo)

- 마지막 입력 동작을 되돌리기
- 여러 단계 undo 지원 (스택 기반)
- 메모 입력/삭제도 undo 대상

### 3.8 숫자 패드 동작

- 1~9 숫자 버튼 표시
- 해당 숫자가 보드에 9개 모두 배치되면 해당 숫자 버튼 사라짐 (또는 비활성화)
- 셀 선택 → 숫자 탭 순서 (기본)
- 숫자 우선 입력 모드: 숫자 길게 눌러 잠금 → 여러 셀에 연속 입력 (설정)

## 4. 화면별 상세 기능

### 4.1 메인 화면

| 요소 | 동작 |
|------|------|
| 일일 도전 카드 | 탭 → 일일 도전 화면 이동 |
| 게임 계속하기 | 탭 → 저장된 게임 재개 |
| 새 게임 | 탭 → 난이도 선택 모달 표시 |
| 🔥 스트릭 | 연속 플레이 일수 표시 |
| 통산 최고 점수 | 트로피 아이콘 + 점수 표시 |

### 4.2 난이도 선택 (모달/시트)

- 쉬움 / 보통 / 어려움 / 전문가 / 마스터
- 선택 시 해당 난이도의 새 퍼즐 생성 후 게임 화면 진입

### 4.3 게임 플레이 화면

| 요소 | 동작 |
|------|------|
| < (뒤로가기) | 메인으로 복귀 (게임 상태 자동 저장) |
| 점수 | 현재 점수 표시 |
| ⚙ 설정 | 설정 화면으로 이동 |
| ⏸ 일시정지 | 타이머 정지, 보드 가림 |
| 셀 탭 | 셀 선택 + 하이라이트 |
| 숫자 버튼 탭 | 선택된 셀에 숫자 입력 |
| 실행 취소 | 마지막 동작 되돌리기 |
| 지우기 | 선택된 셀의 숫자/메모 삭제 |
| 메모 토글 | 메모 모드 ON/OFF |
| 힌트 | 스마트 힌트 표시 |

### 4.3.1 키보드 입력

게임 플레이 화면에서 키보드 입력을 지원합니다:

| 키 | 동작 |
|------|------|
| 화살표 키 (←↑→↓) | 셀 이동 |
| 1-9 숫자 키 | 선택된 셀에 숫자 입력 |
| Backspace / Delete | 선택된 셀의 숫자/메모 삭제 |
| N | 메모 모드 토글 |
| Ctrl+Z | 실행 취소 |
| H | 힌트 |
| Escape | 선택 해제 |

### 4.4 게임 완료 화면

- 모든 셀 채워질 때 트리거
- **완료 애니메이션**: 셀들이 파란색으로 물들어가는 웨이브 효과
- **축하 화면**: 컨페티(confetti) 파티클 애니메이션
- **점수 표시**: 최종 점수, 최고 점수 갱신 여부
- **통계**: 오늘/이번주/이번달/통산 최고 점수
- **기록 갱신 메시지**: "쉬움 난이도에서 최고 시간을 경신했습니다: 09:30!"
- **버튼**: 새 게임, 메인

### 4.5 일일 도전

- 매일 새로운 퍼즐 제공 (날짜 기반 시드)
- 완료 시 해당 날짜에 트로피 마크
- 월별 트로피 수집 현황 (예: 0/28)
- 달력 UI로 진행 상황 시각화

### 4.6 통계 화면

난이도별 탭으로 분류:
- **게임**: 시작한 게임, 승리한 게임, 승률, 실수 없이 승리
- **시간**: 최고 시간, 평균 시간
- **연승**: 현재 연승 기록, 최고 연승 기록
- **최고 점수**: 오늘, 이번 주, 이번 달, 통산
- 통계 초기화 기능

### 4.7 어워드 화면

- 연도별 트로피 표시
- 월별 트로피 카운트
- 도전 탭

### 4.8 설정 화면

| 설정 항목 | 기본값 | 설명 |
|----------|--------|------|
| 사운드 | ON | 효과음 재생 |
| 진동 | ON | 모바일 진동 피드백 |
| 자동 잠금 | OFF | 화면 자동 잠금 방지 |
| 타이머 | ON | 타이머 표시 |
| 점수 애니메이션 | ON | 각 이동으로 얻는 점수 표시 |
| 통계 메시지 | ON | 레벨 완료 사람 수 메시지 |
| 스마트 힌트 | ON | 교육용 힌트 |
| 숫자 우선 입력 | OFF | 숫자 길게 눌러 잠금 |
| 실수 한도 | ON | 게임당 실수 3회 제한 |
| 다크 모드 | OFF | 다크 테마 적용 |

### 4.9 플레이 방법 (Tutorial)

- 3단계 스와이프 가이드
- 1단계: 셀 선택 후 숫자 입력 설명
- 2단계: (추가 규칙)
- 3단계: 메모 모드 사용법
- 건너뛰기 버튼
- 좌/우 네비게이션 화살표
- 페이지 인디케이터 (dots)

### 4.10 규칙 화면

스도쿠 규칙을 설명하는 정적 콘텐츠 화면:
- **9x9 그리드**: 스도쿠 퍼즐은 9x9 격자로 구성됨
- **행/열/블록 규칙**: 각 행, 열, 3x3 블록에 1~9 숫자가 한 번씩만 들어가야 함
- **빈 칸 채우기**: 주어진 숫자를 단서로 빈 칸을 채우는 것이 목표

> 구현: `index.html`에 정적 HTML로 구현됨 (별도 JS 파일 없음)

## 5. 애니메이션 상세

### 5.1 셀 입력 애니메이션
- 숫자 입력 시 약간의 팝(scale up → scale down) 효과
- 점수 "+250" 플로팅 텍스트 (위로 올라가며 페이드아웃)

### 5.2 게임 완료 애니메이션
- 셀들이 순차적으로 파란색 배경으로 전환 (웨이브)
- 각 셀이 약간 확대되었다 원래로 돌아오는 효과
- 전체 완료 후 confetti 파티클
- 점수 카운트업 애니메이션

### 5.3 셀 선택 하이라이트

## 9. 반응형 디자인

### 9.1 뷰포트 기준
- **모바일 우선** (Mobile First)
- 기준 해상도: 884 x 1916 (영상 기준, 약 9:19.5 비율)
- 실제 대응 범위: 320px ~ 428px (너비), 높이 자동

### 9.2 그리드 사이즈 계산
```
그리드 너비 = min(뷰포트 너비 - 패딩, 뷰포트 높이 * 0.45)
셀 크기 = 그리드 너비 / 9
```

### 9.3 데스크톱 대응
- 최대 너비 428px 컨테이너 (모바일 앱 느낌 유지)
- 가운데 정렬
- 배경 색상으로 좌우 여백 채움

## 11. 성능 최적화 고려사항

1. **DOM 업데이트 최소화**: 변경된 셀만 업데이트
2. **이벤트 위임**: 그리드에 단일 이벤트 리스너 (셀별 X)
3. **CSS 애니메이션**: JS 대신 CSS transition/animation 사용
4. **requestAnimationFrame**: confetti 등 복잡한 애니메이션에 사용
5. **사운드 프리로드**: Web Audio API로 사운드 버퍼 미리 로드
6. **터치 최적화**: touch-action, passive 이벤트 리스너
7. **지연 로드**: 설정/통계 등 부수 화면은 필요시 로드
