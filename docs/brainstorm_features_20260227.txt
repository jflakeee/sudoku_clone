1. 스도쿠 변형 규칙 (Variant Rules)
================================================================================

1-1. 킬러 스도쿠 (Killer Sudoku)
────────────────────────────────
  개요:
    - 스도쿠 + 카쿠로의 결합. 표준 스도쿠 규칙에 더해, 점선으로 표시된
      "케이지(cage)" 내 숫자들의 합이 지정된 값과 같아야 함
    - 케이지 내 숫자 중복 불가
    - 산술 능력 + 논리 추론을 동시에 요구
  구현 복잡도: 중
    - 케이지 정의 데이터 구조 추가
    - 케이지 합계 검증 로직
    - 케이지 시각화 (점선 테두리, 합계 표시)
    - 퍼즐 생성기 확장 필요
  사용자 임팩트: 상
    - 클래식 스도쿠 다음으로 가장 인기 있는 변형
    - 중급 이상 사용자의 리텐션 크게 향상
  참고 사례:
    - Cracking the Cryptic (Killer Sudoku CTC 앱)
    - Sudoku.com (킬러 모드 지원)
    - Wikipedia: https://en.wikipedia.org/wiki/Killer_sudoku

1-2. 대각선 스도쿠 (X-Sudoku / Diagonal Sudoku)
────────────────────────────────────────────────
  개요:
    - 표준 규칙 + 두 대각선(좌상-우하, 우상-좌하)에도 1~9가 한 번씩 등장해야 함
    - X자 형태의 추가 제약으로 난이도 상승
  구현 복잡도: 하
    - 대각선 검증 로직만 추가 (2개 대각선)
    - 대각선 시각적 표시 (배경색 또는 선)
    - 기존 생성기에 대각선 제약만 추가
  사용자 임팩트: 중
    - 구현 대비 효과 좋음 (최소 노력, 최대 다양성)
    - 클래식에 익숙한 사용자에게 신선한 도전
  참고 사례:
    - SudokuBliss: https://sudokubliss.com/guides/types-of-sudoku-puzzles
    - Sudoku Variants by Logic Wiz 앱

1-3. 지그소 스도쿠 (Jigsaw / Irregular Sudoku)
──────────────────────────────────────────────
  개요:
    - 정사각형 3x3 블록 대신 불규칙한 퍼즐 조각 모양의 영역을 사용
    - 각 불규칙 영역에 1~9가 한 번씩 등장
    - 시각적으로 독특하고 풀이 방식이 다름
  구현 복잡도: 상
    - 불규칙 영역 정의 및 시각화 시스템 필요
    - 퍼즐 생성 알고리즘 대폭 수정 필요
    - 영역 랜덤 생성 또는 프리셋 영역 세트 필요
  사용자 임팩트: 중~상
    - 시각적 신선함이 크고 차별화 요소
    - 난이도 조절이 다소 어려움
  참고 사례:
    - Sudoku Studio: https://www.sudokustudio.app/variants
    - Sudoku Variations Puzzle Book (Amazon)

1-4. 하이퍼 스도쿠 (Hyper Sudoku / Windoku)
───────────────────────────────────────────
  개요:
    - 표준 9개 블록 외에 4개의 추가 3x3 영역(창문 모양)에서도 1~9 규칙 적용
    - 추가 영역 위치: (2,2)-(4,4), (2,6)-(4,8), (6,2)-(8,4), (6,6)-(8,8)
  구현 복잡도: 하
    - 4개 추가 영역 검증 로직
    - 추가 영역 시각적 강조 (배경색)
    - 생성기에 추가 제약 반영
  사용자 임팩트: 중
    - 대각선 스도쿠와 비슷한 수준의 변형
    - 해외에서 인기 있으나 한국에서는 인지도 낮음
  참고 사례:
    - Sudoku Times: https://sudokutimes.com/top-sudoku-variants/
    - eev.ee variant reference: https://eev.ee/fyi/variant-sudoku/

1-5. 서모 스도쿠 (Thermo Sudoku)
────────────────────────────────
  개요:
    - 그리드에 온도계 모양의 선이 그려짐
    - 온도계 방향(구 -> 끝)으로 숫자가 순증가해야 함
    - 시각적이고 직관적인 추가 제약
  구현 복잡도: 중
    - 서모 라인 렌더링 (구 + 선)
    - 순증가 검증 로직
    - 서모 퍼즐 생성 알고리즘
  사용자 임팩트: 중~상
    - Cracking the Cryptic 유튜브로 인기 급상승 중
    - 시각적 매력이 높고 풀이가 직관적
  참고 사례:
    - Cracking the Cryptic (Thermo Sudoku)
    - GM Puzzles: https://www.gmpuzzles.com/blog/category/sudoku/thermo-sudoku/
    - Mastering Sudoku: https://masteringsudoku.com/thermo-sudoku/

1-6. 샌드위치 스도쿠 (Sandwich Sudoku)
─────────────────────────────────────
  개요:
    - 그리드 외곽에 숫자 표시
    - 각 행/열에서 1과 9 사이에 "끼인" 숫자들의 합을 나타냄
    - 예: 외곽 숫자 0 = 1과 9가 인접, 4 = 사이에 4 하나만 존재
  구현 복잡도: 중
    - 외곽 숫자 표시 UI
    - 샌드위치 합계 검증 로직
    - 퍼즐 생성 알고리즘
  사용자 임팩트: 중
    - 독특한 풀이 방식으로 매니아층 형성
    - 일반 사용자에게는 진입 장벽 있음
  참고 사례:
    - Cracking the Cryptic (Sandwich Sudoku)
    - DKM Games: https://dkmgames.com/SandwichSudoku/

1-7. 화살 스도쿠 (Arrow Sudoku)
──────────────────────────────
  개요:
    - 원(circle) 셀에서 화살표가 나감
    - 화살표 경로 위의 숫자 합 = 원 안의 숫자
    - 원과 화살표가 시각적으로 명확한 힌트 제공
  구현 복잡도: 중
    - 원 + 화살표 렌더링
    - 합계 검증 로직
    - 퍼즐 생성
  사용자 임팩트: 중
    - 킬러 스도쿠와 유사한 산술 요소
    - 시각적 표현이 더 직관적
  참고 사례:
    - Cracking the Cryptic (Arrow Sudoku 앱)
    - Apple App Store: https://apps.apple.com/us/app/arrow-sudoku/id1568407537

1-8. 포그 오브 워 스도쿠 (Fog of War Sudoku)  *** 추천 ***
─────────────────────────────────────────────
  개요:
    - 그리드가 안개로 덮인 상태에서 시작
    - 올바른 숫자를 배치하면 주변 안개가 걷히며 새로운 단서 노출
    - 디지털 전용 변형 (종이에서는 불가능)
    - 초반 풀이가 선형적이다가 점차 자유도 증가
  구현 복잡도: 중
    - 안개 시각 효과 (CSS blur/opacity)
    - 안개 해제 로직 (정답 입력 시 인접 셀 공개)
    - 기존 퍼즐에 안개 레이어만 추가 가능
  사용자 임팩트: 상
    - 웹/디지털 전용이므로 차별화 극대화
    - 탐험 느낌의 게임성 부여 (Discovery 재미)
    - Cracking the Cryptic에서 대인기 → Kickstarter 캠페인까지 진행
  참고 사례:
    - Cracking the Cryptic - Fog of War (Steam)
    - SudokuPad: https://sudokupad.app
    - Kickstarter 캠페인 진행됨

2. 게임화(Gamification) 요소
================================================================================

2-1. 시즌제 / 배틀패스 시스템
────────────────────────────
  개요:
    - 4~6주 단위 시즌으로 운영
    - 매일 미션 수행 → 경험치(XP) 획득 → 시즌 레벨 상승
    - 무료 트랙 + 프리미엄 트랙 (이중 보상 체계)
    - 시즌별 테마/스킨 보상
  구현 복잡도: 상
    - 시즌 관리 시스템
    - 미션/퀘스트 엔진
    - 보상 체계 및 인벤토리
    - 시즌 타이머 및 진행률 UI
  사용자 임팩트: 상
    - DAU 40~60% 향상 효과 (업계 통계)
    - 장기 리텐션 극대화
    - 수익화 가능 (프리미엄 패스)
  참고 사례:
    - GameRefinery: https://www.gamerefinery.com/battle-pass-trend-mobile-games/
    - GameAnalytics: https://www.gameanalytics.com/blog/designing-battle-passes-in-mobile-games
    - Solitaire Home Design (퍼즐 게임 배틀패스 사례)

2-2. 레벨업 / 경험치 시스템
──────────────────────────
  개요:
    - 게임 완료 시 경험치 획득 (난이도, 시간, 정확도 기반)
    - 레벨업 시 타이틀/칭호 변경 (초보자 → 견습생 → 스도쿠 마스터)
    - 레벨에 따라 새로운 기능/변형 모드 언락
  구현 복잡도: 하~중
    - XP 계산 로직 (기존 scorer.js 확장)
    - 레벨 테이블 정의
    - 레벨 표시 UI 및 레벨업 연출
  사용자 임팩트: 중~상
    - 성장감(Progression) 부여
    - 일일 활성 유지에 효과적
  참고 사례:
    - Sudoku Slash: https://sudokuslash.com/
    - Yu-kai Chou 게이미피케이션 프레임워크

2-3. 스트릭 강화 시스템
────────────────────────
  개요:
    - 현재 스트릭 배지 구현됨 → 이를 강화
    - 연속 일수에 따른 보상 차등 (7일: 브론즈, 30일: 실버, 100일: 골드)
    - 스트릭 보호권 (1일 놓쳐도 유지)
    - 스트릭 마일스톤 달성 시 특별 보상
  구현 복잡도: 하
    - 기존 스트릭 로직 확장
    - 보상 체계 추가
    - 스트릭 보호 메커니즘
  사용자 임팩트: 중~상
    - 일일 복귀율 직접 향상
    - 손실 회피(Loss Aversion) 심리 활용
  참고 사례:
    - Duolingo 스트릭 시스템
    - Yu-kai Chou: https://yukaichou.com/gamification-study/

2-4. 테마 / 스킨 시스템
──────────────────────
  개요:
    - 그리드 스킨: 나무 질감, 네온, 미니멀, 빈티지 등
    - 숫자 폰트 변경: 필기체, 픽셀, 고전 등
    - 배경 테마: 자연, 우주, 도시 등
    - 완료 시 이펙트 커스터마이징
  구현 복잡도: 하~중
    - CSS 변수 기반 테마 시스템 (이미 다크모드 구현됨)
    - 테마 프리셋 JSON 정의
    - 테마 선택 UI
  사용자 임팩트: 중
    - 개인화 욕구 충족
    - 보상 체계와 연계 시 시너지
  참고 사례:
    - Sudoku Dragon 커스터마이징
    - 다수 모바일 게임의 테마 시스템

2-5. 업적(Achievement) 시스템 확장
─────────────────────────────────
  개요:
    - 현재 어워드(트로피/도전) 구현됨 → 확장
    - 숨겨진 업적 (Hidden Achievement)
    - 희귀도 등급 (일반/희귀/전설)
    - 업적 포인트 누적 → 보상
    - 예시 업적:
      * "번개" - 5분 내 마스터 난이도 완료
      * "완벽주의자" - 메모 없이 50게임 연속 완료
      * "탐험가" - 모든 변형 모드 1회 이상 플레이
      * "포그 헌터" - 포그 오브 워 모드 30회 완료
  구현 복잡도: 중
    - 업적 조건 감지 시스템
    - 업적 알림 UI
    - 프로필 페이지 업적 표시
  사용자 임팩트: 중~상
    - 목표 지향적 플레이 촉진
    - 컨텐츠 탐색 유도
  참고 사례:
    - Sudoku.com 어워드 시스템: https://sudoku.com/awards
    - Xbox/PlayStation 업적 시스템 패턴

3. 소셜/경쟁 기능
================================================================================

3-1. 실시간 1:1 대전 (PvP Battle)
────────────────────────────────
  개요:
    - 두 플레이어가 동일한 퍼즐을 동시에 풀기
    - 실시간 진행률 표시 (상대방 몇 % 완료)
    - 승리 조건: 먼저 완료 or 제한 시간 내 더 많이 채우기
    - 매칭 시스템: ELO 기반 레이팅
  구현 복잡도: 상
    - WebSocket 서버 필요
    - 매칭 시스템
    - 실시간 동기화
    - 안티치트 메커니즘
  사용자 임팩트: 상
    - 경쟁 요소로 활성 사용자 크게 증가
    - 바이럴 효과 기대
  참고 사례:
    - SudokuRace: https://www.sudokurace.io/
    - Sudoku Slash: https://sudokuslash.com/play-with-friends/
    - Sudoku Battles: https://www.sudokubattles.com/

3-2. 토너먼트 / 대회 시스템
──────────────────────────
  개요:
    - 주간/월간 토너먼트
    - 참가자 동일 퍼즐 세트 풀기 → 총점으로 순위
    - 토너먼트 등급: 브론즈/실버/골드/다이아몬드
    - 시상 및 메달 시스템
  구현 복잡도: 상
    - 토너먼트 스케줄링 시스템
    - 공정성 보장 (동일 퍼즐 분배)
    - 순위 계산 및 보상 분배
    - 서버 인프라 필요
  사용자 임팩트: 상
    - 이벤트성 컨텐츠로 집중도 향상
    - 커뮤니티 형성 촉진
  참고 사례:
    - Sudoku.com 토너먼트: https://sudoku.com/tournaments
    - Sudoku Slash 토너먼트 시스템

3-3. 친구 초대 & 도전장
────────────────────────
  개요:
    - 링크 공유로 친구에게 도전
    - "이 퍼즐을 풀어봐!" → 공유 링크 생성
    - 친구의 기록과 내 기록 비교
    - 프라이빗 방 생성 (방 코드 입력)
  구현 복잡도: 중
    - 공유 링크/코드 생성 시스템
    - 결과 비교 페이지
    - 방 시스템 (서버 필요하나 P2P로 간소화 가능)
  사용자 임팩트: 중~상
    - 유기적 사용자 유입 채널
    - 소셜 바이럴 효과
  참고 사례:
    - SudokuFriends: https://www.sudokufriends.io/
    - UsDoku: https://www.usdoku.com/

3-4. 글로벌 리더보드
──────────────────
  개요:
    - 일간/주간/월간/통산 리더보드
    - 난이도별 분류
    - 백분위 표시 ("상위 5% 솔버")
    - 국가/지역별 랭킹
  구현 복잡도: 중
    - 리더보드 서버 API
    - 익명/닉네임 시스템
    - 랭킹 계산 로직

4. 교육/연습 모드

4-1. 기법별 연습 모드 (Technique Practice)
────────────────────────────────────────
  개요:
    - 스도쿠 풀이 기법을 단계별로 학습
    - 기법 카테고리:
      * 초급: Naked Single, Hidden Single
      * 중급: Naked Pair/Triple, Pointing Pair, Box/Line Reduction
      * 고급: X-Wing, Swordfish, XY-Wing, Coloring
      * 전문가: Almost Locked Sets, Forcing Chains
    - 각 기법에 해당하는 미니 퍼즐 제공
    - 기법 적용 위치 힌트 제공
  구현 복잡도: 상
    - 기법 감지 알고리즘 (solver.js 대폭 확장)
    - 기법별 미니 퍼즐 생성/큐레이션
    - 기법 해설 UI
    - 단계적 난이도 조절
  사용자 임팩트: 상
    - 교육적 가치 극대화
    - 초보자 → 고수 성장 경로 제공
    - 차별화 포인트 (대부분의 앱이 미흡)
  참고 사례:
    - Sudoku Coach: https://sudoku.coach/en/home
    - Sudoku Custom (30+ 기법 튜토리얼)
    - Hintoku: https://apps.apple.com/us/app/hintoku-your-sudoku-coach/id6744828400

4-2. 스마트 힌트 해설 (Intelligent Hint System)
──────────────────────────────────────────────
  개요:
    - 현재: 힌트 = 정답 직접 표시
    - 개선: 다단계 힌트 시스템
      * 1단계: "이 행을 살펴보세요" (방향만 제시)
      * 2단계: "이 셀에 들어갈 수 있는 후보 확인해보세요" (영역 지시)
      * 3단계: "Naked Pair 기법을 적용해보세요" (기법 이름)
      * 4단계: 구체적 풀이 과정 시각화
      * 5단계: 정답 공개
  구현 복잡도: 상
    - 풀이 과정 분석 엔진
    - 기법 감지 및 설명 생성
    - 단계별 UI 구현
  사용자 임팩트: 상
    - 학습 도구로서의 가치 극대화
    - 힌트 사용에 대한 죄책감 감소
    - Enjoy Sudoku의 핵심 차별화 기능
  참고 사례:
    - Enjoy Sudoku (다단계 힌트): http://www.enjoysudoku.com/
    - Sudoku Tutor 앱
    - Sudoku Solver: https://sudokusolver.app/

4-3. 풀이 과정 리플레이
────────────────────────
  개요:
    - 완료된 퍼즐의 풀이 과정을 타임라인으로 재생
    - 속도 조절 가능 (1x, 2x, 4x)
    - 각 단계별 사용된 기법 표시
    - 자신의 풀이 vs 최적 풀이 비교
  구현 복잡도: 중
    - 풀이 기록 저장 (history.js 확장)
    - 리플레이 플레이어 UI
    - 기법 분석 로직 (옵션)
  사용자 임팩트: 중
    - 자기 개선 도구
    - 교육 콘텐츠 활용 가능
  참고 사례:
    - 체스 게임 리플레이 시스템
    - SudokuPad 기록 기능

6-1. PWA 완전 지원
─────────────────

  구현 복잡도: 중

    - 캐시 전략 설계 (Cache First for assets, Network First for data)
    - 오프라인 퍼즐 사전 생성 및 저장
    - 온라인 복귀 시 데이터 동기화

    - 오프라인 플레이 가능 (통근/비행기)
    - 설치 없이 앱처럼 사용
    - PWA 시장 2024년 $20.8B → 2033년 $212.4B 예상
  참고 사례:
    - 2024~2026 PWA 트렌드

  구현 복잡도: 상
    - 백엔드 서버/DB 필요
    - OAuth 연동
    - 충돌 해결 로직 (Conflict Resolution)
    - 개인정보 보호 정책
  사용자 임팩트: 중~상
    - 기기 교체/분실 시 데이터 보존

  참고 사례:

핵심 인사이트 요약
================================================================================

  1. 포그 오브 워 스도쿠는 웹 전용 변형으로 종이에서 불가능한 경험을 제공하며
     Cracking the Cryptic에서 대인기를 끌고 있어 최우선 추천 변형임

  2. 대각선/하이퍼 스도쿠는 구현 비용 대비 효과가 가장 높은 Quick Win 항목

  3. 시즌제+스트릭 조합은 DAU 40~60% 향상 효과가 입증된 게이미피케이션 패턴

  4. 다단계 힌트 시스템은 Enjoy Sudoku의 핵심 차별화 요소이며, 교육적 가치와
     사용자 만족도를 동시에 높이는 기능

  5. PWA 완전 지원은 웹 앱의 네이티브 격차를 줄이는 핵심 기술 요소

  6. 소셜/경쟁 기능은 백엔드 인프라가 필요하므로 서버리스(Firebase/Supabase)
     활용을 검토하여 개발 복잡도를 낮출 것을 권장
